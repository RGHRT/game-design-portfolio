# RPG 角色成长与关卡难度数值表

## 项目简介

本项目用于展示基础数值策划能力。

我设计了一套简化的 RPG 角色成长与关卡难度数值表，用于模拟角色属性成长、经验消耗、敌人强度、伤害公式、关卡推荐等级和资源产出消耗之间的关系。

这不是某个具体商业游戏的真实数值，而是一个用于作品集展示的虚构数值模型。它的重点不在于“数值一定完美”，而在于展示我能够使用 Excel 建立公式驱动的数值表，并从成长节奏、资源压力和关卡体验角度进行分析。

相比单纯写“我了解数值策划”，这份表格希望展示的是一种更具体的能力：能够把角色成长、敌人强度、关卡难度和资源消耗放在同一个系统中观察，并尝试理解它们之间如何相互影响玩家体验。

## 0_设计说明

这一页用于说明整套数值表的设计目标和基础参数。

主要内容包括：

- 基础生命值
- 基础攻击力
- 基础防御力
- 属性成长率
- 经验成长率
- 技能倍率
- 克制系数
- 伤害保底比例
- 敌人倍率
- 关卡难度评价规则

这一页的作用是让整个表格不是一堆孤立数字，而是由基础参数驱动的数值系统。后续角色属性、敌人属性、伤害公式和关卡难度，都会围绕这些参数展开。

### 设计目的

我希望通过这张表模拟一个简化 RPG 游戏中的基础成长逻辑：

玩家在前期获得较快成长反馈；  
中期开始感受到经验和资源压力；  
后期成长速度放缓，但角色强度仍持续提升；  
关卡难度随着玩家等级逐步上升，并在关键节点提供一定挑战。

## 1_角色属性成长

这一页设计角色 1—80 级的基础属性成长。

主要字段包括：

- 等级
- 生命值 HP
- 攻击力 ATK
- 防御力 DEF
- 战力评分
- 升级所需经验
- 累计经验

### 设计思路

角色成长不能完全线性。如果每一级提升都一样，前期反馈可能不够明显，后期也容易缺少成长感。因此这张表中采用了带成长率的属性曲线，让角色随着等级提升逐步变强。

前期成长速度相对较快，目的是让玩家在刚开始培养角色时能够明显感受到提升。中后期成长逐步放缓，主要用于延长长期培养目标，避免玩家太快达到完全成型状态。

### 策划角度理解

角色成长表不是单纯决定“角色有多强”，而是在设计玩家每次升级后的反馈感。

如果前期升级反馈太弱，玩家会觉得培养没有意义；  
如果后期成长过快，数值容易膨胀；  
如果成长过慢，玩家又会觉得投入资源没有回报。  

因此，角色成长曲线需要在“爽感”和“长期目标”之间保持平衡。

## 2_经验消耗曲线

这一页主要观察升级经验和累计经验的变化。

主要字段包括：

- 等级
- 升级所需经验
- 累计经验
- 阶段划分
- 阶段体验说明

### 阶段划分

| 阶段   | 等级范围 | 设计目标                           |
| ------ | -------- | ---------------------------------- |
| 前期   | 1—20 级  | 快速升级，给予明显成长反馈         |
| 中期   | 21—50 级 | 提高经验需求，引导玩家开始规划资源 |
| 成型期 | 51—70 级 | 角色逐渐成型，培养成本明显提高     |
| 后期   | 71—80 级 | 成长速度放缓，作为长期培养目标     |

### 设计思路

经验消耗曲线的核心作用，是控制玩家成长速度。

前期经验消耗较低，让玩家可以较快体验角色成长。  
中期开始提高经验成本，让玩家意识到资源需要规划。  
后期经验消耗进一步提高，用于延长角色养成周期，并制造长期目标。  

### 策划角度理解

经验曲线会直接影响玩家对培养节奏的感受。

前期太慢，玩家容易失去兴趣；  
中期太快，玩家很快失去目标；  
后期太陡，玩家会觉得培养压力过大。  

合理的经验曲线应该让玩家持续感到“我在变强”，同时又不会过早完成全部培养。

## 3_敌人属性设计

这一页设计普通怪、精英怪和 Boss 的属性成长。

主要字段包括：

- 敌人类型
- 推荐等级
- 敌方生命值
- 敌方攻击力
- 敌方防御力
- 强度倍率
- 设计说明

### 敌人类型

| 敌人类型 | 设计定位                               |
| -------- | -------------------------------------- |
| 普通怪   | 基础战斗单位，用于提供稳定战斗压力     |
| 精英怪   | 强度高于普通怪，用于制造阶段性压力     |
| Boss     | 关卡核心挑战，通常需要玩家具备一定练度 |

### 设计思路

敌人属性不能简单和玩家属性等比例增长。如果敌人始终过弱，关卡缺少挑战；如果敌人增长过快，玩家会在推荐等级附近也难以通关。

因此我在设计中使用了不同敌人倍率，让普通怪、精英怪、Boss 形成强度层级：

普通怪用于维持基础战斗节奏；  
精英怪用于制造局部压力；  
Boss 用于验证玩家的角色练度和资源投入成果。  

### 策划角度理解

敌人设计不只是填生命值和攻击力，而是在设计关卡体验。

普通怪决定战斗节奏；  
精英怪决定玩家是否需要调整策略；  
Boss 决定关卡是否有记忆点和挑战感。

## 4_伤害公式

这一页构建了一个简化的伤害公式，用于模拟角色面对不同敌人时的输出表现。

### 公式设计

```text
最终伤害 = MAX(攻击力 × 技能倍率 × 克制系数 - 敌方防御, 攻击力 × 伤害保底比例)
```

### 参数含义

| 参数         | 含义                     |
| ------------ | ------------------------ |
| 攻击力       | 角色当前等级下的攻击属性 |
| 技能倍率     | 技能对伤害的放大比例     |
| 克制系数     | 模拟属性克制或职业优势   |
| 敌方防御     | 敌人减伤能力             |
| 伤害保底比例 | 避免伤害被防御完全抵消   |

### 设计思路

这个公式虽然比较简化，但能够体现几个基础问题：

攻击力成长会影响输出提升；  
敌方防御会影响玩家实际伤害感受；  
技能倍率决定技能释放时的爆发效果；  
伤害保底可以避免玩家面对高防御敌人时完全打不动。  

### 策划角度理解

伤害公式是战斗数值的核心之一。它不只是数学计算，还会影响玩家对战斗的直观感受。

如果防御收益过高，玩家可能会感觉自己完全打不动敌人；  
如果攻击收益过高，敌人会变得过脆；  
如果技能倍率过高，普通攻击可能失去存在感；  
如果伤害保底过低，低攻击角色体验会很差。  

因此，伤害公式需要同时考虑数值平衡和玩家感受。

## 5_关卡难度配置

这一页设计 30 个关卡的推荐等级、敌人配置和难度指数。

主要字段包括：

- 关卡编号
- 推荐等级
- 敌人数量
- 是否包含 Boss
- 敌方总 HP
- 玩家战力
- 难度指数
- 难度评价

### 难度评价

| 难度评价 | 含义                           |
| -------- | ------------------------------ |
| 偏易     | 玩家在推荐等级下可以较轻松通过 |
| 适中     | 玩家需要正常培养和基本操作     |
| 偏难     | 玩家需要更高练度或更合理策略   |
| 高压     | 关卡挑战明显，可能需要重点调整 |

### 设计思路

关卡难度不应该只看敌人总 HP，也要结合推荐等级、敌人数量、Boss 配置和玩家战力。

我通过难度指数观察关卡整体压力，希望让关卡形成逐步上升的难度曲线。前期关卡偏简单，用来帮助玩家熟悉系统；中期关卡逐步增加压力；后期关卡则需要玩家具备更完整的角色成长和资源投入。

### 策划角度理解

关卡难度设计的关键不是让玩家一直赢，也不是让玩家一直卡关，而是让玩家在合适的阶段感受到合适的挑战。

过于简单，玩家会觉得培养没有意义；  
过于困难，玩家会觉得数值压迫感过强；  
合理的难度曲线应该让玩家觉得“我需要变强”，并且在变强之后真的能够通过。

## 6_资源产出消耗

这一页按照前期、中期、成型期、后期四个阶段，设计角色培养所需的经验、金币和材料消耗。

主要字段包括：

- 阶段
- 等级范围
- 经验消耗
- 金币消耗
- 材料消耗
- 资源压力评价
- 设计说明

### 阶段性资源压力

| 阶段   | 资源压力 | 设计目的                       |
| ------ | -------- | ------------------------------ |
| 前期   | 低       | 让玩家快速成长，建立正反馈     |
| 中期   | 中       | 引导玩家开始规划培养目标       |
| 成型期 | 较高     | 让玩家围绕核心角色投入资源     |
| 后期   | 高       | 延长长期培养目标，避免过快毕业 |

### 设计思路

资源消耗不能从一开始就很高，否则玩家会被劝退。前期应该让玩家快速体验成长，中期开始逐渐提高资源需求，后期则通过更高成本制造长期目标。

### 策划角度理解

资源消耗设计的本质，是控制玩家的成长节奏和目标密度。

资源过多，玩家很快毕业；  
资源过少，玩家感到挫败；  
资源阶段性变化合理，玩家才会在不同阶段形成不同目标。

## 7_平衡性观察

这一页用于总结当前数值模型的特点、潜在问题和后续优化方向。

### 当前设计特点

1. 角色前期成长较快，能够提供明显正反馈。
2. 中后期经验消耗逐步提高，能够形成长期培养压力。
3. 敌人强度按照普通怪、精英怪、Boss 形成层级。
4. 伤害公式保留防御减伤和伤害保底，避免极端情况。
5. 关卡难度随推荐等级逐步提升，整体符合成长节奏。
6. 资源消耗分阶段递增，有助于形成长期目标。

### 可能存在的问题

1. 当前模型仍然比较简化，没有考虑多角色队伍搭配。
2. 伤害公式没有加入暴击、攻速、技能冷却等复杂因素。
3. 敌人行为模式较简单，主要依靠属性差异体现难度。
4. 关卡难度指数仍然较粗略，需要结合实际战斗模拟进一步验证。
5. 资源产出和消耗的关系还可以更细致，例如加入日常副本、活动奖励、商店兑换等来源。

### 后续优化方向

- 增加多角色队伍战力计算；
- 加入暴击率、暴击伤害、攻速等战斗参数；
- 设计不同职业定位，如坦克、输出、治疗、辅助；
- 增加装备系统或技能升级系统；
- 加入资源产出渠道，观察产出和消耗是否平衡；
- 使用图表展示经验曲线、战力曲线和难度曲线；
- 通过 Python 或 Excel 模拟不同培养策略下的成长速度。

## 我的设计思路总结

这套数值表主要围绕三个问题展开。

第一，角色成长不能过快，否则玩家很快失去长期目标；也不能过慢，否则玩家会感到挫败。前期应当提供快速成长反馈，中后期则逐步提高培养成本。

第二，关卡难度应该随着玩家成长逐步上升，让玩家在推荐等级附近感到有挑战，但不是完全打不过。玩家应该能通过培养、策略或资源投入改善通关体验。

第三，资源消耗需要形成阶段性压力。前期应该给玩家较快反馈，中期开始引导玩家规划资源，后期则通过更高成本延长长期培养目标。

从这次表格设计中，我认识到数值策划不是孤立地填属性、填经验、填怪物血量，而是要把玩家成长、敌人强度、资源压力和关卡体验放在一起考虑。
